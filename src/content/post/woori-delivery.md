## 개요

우리동네GS 앱에 택배 서비스를 신규로 연동하는 프로젝트의 백엔드를 설계·개발했습니다.
앱 인증 흐름 설계부터 주소록·쿠폰 기능 개발, 오프쇼어 개발팀과의 연동까지 전반을 담당했습니다.

- **회사**: GS리테일 / 물류DX팀
- **기간**: 2023.11 – 2024.12
- **역할**: 백엔드 설계·개발, 인증 흐름 설계, 오프쇼어 협업
- **스택**: Java, Spring Boot 3, JWT, KMS, Kubernetes, Aurora PostgreSQL, Datadog

---

## 배경

우리동네GS 앱은 기존 세션 기반 인증을 사용하고 있었습니다.
택배 서비스를 연동하면서 앱·웹뷰·단말 등 다양한 접근 경로에서 일관된 인증 흐름이 필요했고,
확장성과 보안을 모두 고려한 인증 방식으로의 전환이 필요한 시점이었습니다.

---

## 주요 기여

### 1. RSA(KMS) 기반 JWT 인증 설계

기존 세션 인증 방식에서 JWT로 전환하며, 키 노출 위험을 낮추기 위해 AWS KMS를 이용한 RSA 기반 구조를 설계했습니다.

- **공개키 검증 방식**: 수신 측이 공개키만으로 토큰을 검증하므로, 시크릿 키를 공유하지 않아도 됩니다.
- **확장성**: 단말 수나 서버가 늘어도 인증 흐름이 변경되지 않습니다.

```
[앱 / 단말]
    │  로그인 요청
    ▼
[인증 서버] ── KMS에서 RSA 서명 ──▶ JWT 발급
    │
    ▼
[서비스 서버] ── 공개키로 검증 ──▶ 인가 처리
```

### 2. 웹뷰 인증 연속성 처리

웹뷰에서 앱 세션이 끊기는 문제를 방지하고자, 세션 흐름에 JWT를 접목해 인증이 유지되도록 처리했습니다.
앱과 웹뷰 간 컨텍스트 전환 시 토큰이 자연스럽게 이어지도록 설계했습니다.

### 3. 주소록 / 쿠폰 기능 개발

택배 서비스 이용에 필요한 핵심 기능을 구현했습니다.

- **주소록**: 배송지 등록·수정·삭제·기본 설정
- **쿠폰**: 등록·조회·사용·선물 전체 플로우

### 4. 오프쇼어 팀 연동 조율

우리동네GS 앱 개발을 담당하는 오프쇼어 팀과의 API 연동 과정에서, 스펙 차이·타임존·인코딩 등 다양한 연동 이슈를 직접 조율했습니다.

- 이슈 발생 시 재현 케이스를 문서화해 빠른 합의를 이끌어냈습니다.
- 해외 팀과의 비동기 커뮤니케이션 경험을 쌓았습니다.

## 결과

- RSA(KMS) 기반 JWT 도입으로 단말·웹뷰·앱 전반의 인증 흐름 통합
- 주소록·쿠폰 기능 정상 오픈, 서비스 안정적 운영
- 오프쇼어 팀과의 연동 이슈를 조율하며 일정 내 출시 완료

## 회고

세션에서 JWT로 전환하는 작업은 단순히 토큰을 발급하는 것이 아니라, 다양한 클라이언트 환경에서 인증이 끊기지 않도록 흐름 전체를 설계하는 작업이었습니다.
RSA(KMS) 구조를 선택한 이유는 키 노출 리스크를 구조적으로 줄이기 위해서였고, 이 결정이 이후 유사 서비스 인증 설계의 기준이 됐습니다.
